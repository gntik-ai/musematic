# Reasoning Traces

Reasoning traces explain how the reasoning engine evaluated branches, selected approaches, and reached convergence. They are designed for inspection, not for blind trust.

Look for branch summaries, budget use, policy checks, confidence signals, and self-correction events. If a trace is missing, the workflow may have used a non-reasoning path or the caller may not have visibility.

Do not paste sensitive trace content into external tools. Treat traces as workspace data and follow the same retention and disclosure rules used for execution artifacts.
