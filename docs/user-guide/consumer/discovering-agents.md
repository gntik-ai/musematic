# Discovering Agents

Consumers begin in the marketplace or workspace catalog. Search by capability, FQN, owner, trust status, tags, and visibility. Results respect zero-trust visibility rules, so unavailable agents are hidden rather than shown as disabled.

Open an agent detail page before invoking it. Check the purpose statement, trust signals, current revision, known limitations, pricing or budget notes, and certification state. If an agent is part of a fleet, review the fleet policy because routing can affect cost, latency, and explainability.

When no agent appears for a task, ask a workspace admin to review visibility grants or ask a creator to publish an appropriate revision.
