# Alerts

Alerts notify consumers when a workflow needs attention, an execution fails, a budget threshold is crossed, or a platform incident affects their workspace.

Open the alert detail before retrying the action. The detail should identify the affected workspace, goal, execution, or integration, and should link to the relevant trace or runbook. Dismissing an alert only clears the notification; it does not change execution state.

For repeated alerts, ask an admin or operator to inspect policy, capacity, provider limits, or incident history.
