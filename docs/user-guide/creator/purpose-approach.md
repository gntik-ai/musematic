# Purpose and Approach

The purpose statement explains what the agent is allowed to do. The approach explains how it does that work, including data sources, tools, models, safety checks, and expected outputs.

Write purpose narrowly enough for policy review. For example, "summarize approved support tickets for weekly reporting" is easier to govern than "analyze support data." Include limitations and human-review expectations.

Trust reviewers use this material during certification, and consumers use it before invoking the agent.
