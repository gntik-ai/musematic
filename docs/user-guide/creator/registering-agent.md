# Registering an Agent

Creators register agents in the catalog with ownership, namespace, FQN, purpose, package metadata, and visibility. The initial profile can be private while the agent is tested.

Before registration, confirm that the namespace reflects the team or product boundary and that the owner can maintain the package. Add enough metadata for search, certification, and operational support.

Registration does not guarantee broad availability. Visibility grants, certification, and policy attachments determine where the agent can be discovered and invoked.
