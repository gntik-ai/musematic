# Compliance Mapping

Security-compliance evidence maps operational events and release artifacts into common frameworks. This page is an orientation guide, not a certification claim.

| Framework | Musematic Evidence Source |
| --- | --- |
| SOC 2 | Audit chain, access reviews, incident records, change evidence, availability dashboards. |
| ISO 27001 | Risk controls, vulnerability gates, access policy, supplier/integration controls, backup tests. |
| GDPR | DSR workflows, consent history, residency policy, DLP events, signed deletion tombstones. |
| HIPAA | Access controls, audit logs, encryption, incident records, minimum necessary data handling. |
| PCI DSS | Segmented network policy, vulnerability scanning, access review, secrets handling. |

Feature 074 and the UPD-024 audit-chain work provide the evidence substrate. Manual evidence should include reviewer, control mapping, source hash, date range, and retention policy.
