# v1.3.0

Version 1.3.0 groups the UPD-036, UPD-037, UPD-038, and UPD-039 cohort: admin and super-admin workbench completion, public signup flow, multilingual repository README work, and the comprehensive documentation site.

## Highlights

- Admin and super-admin workflows cover identity, tenancy, OAuth, lifecycle, cost, observability, integrations, multi-region operations, and audit logs.
- Public signup adds verification, approval, OAuth callback handling, and profile completion state.
- Repository README localization adds canonical English source, translation parity checks, and weekly external-link validation.
- Documentation site adds the FR-605 eleven-section MkDocs structure, generated OpenAPI, generated environment variable reference, generated Helm values reference, runbooks, installation guides, security guide, and localization scaffolding.
- HashiCorp Vault integration adds `mock`, `kubernetes`, and `vault` secret modes, the optional Vault Helm chart with `dev`, `standalone`, and `ha` presets, migration tooling, admin status endpoints, and Vault observability.
- OAuth bootstrap adds `PLATFORM_OAUTH_GOOGLE_*` and `PLATFORM_OAUTH_GITHUB_*` install-time configuration, Vault-backed client-secret seeding, provider source badges, rotation, reseed, role-mapping, history, rate-limit, and connectivity controls.

## Compatibility

REST APIs remain under `/api/v1`. The documentation build now treats generated API, environment, Helm, and FR-reference outputs as drift-checked artifacts. Chart value changes should include regenerated per-chart README files and the aggregate Helm values page.

## Vault Integration Notes

New environment variables use the `PLATFORM_VAULT_*` prefix, including `PLATFORM_VAULT_MODE`, `PLATFORM_VAULT_ADDR`, `PLATFORM_VAULT_AUTH_METHOD`, `PLATFORM_VAULT_TOKEN`, `PLATFORM_VAULT_APPROLE_ROLE_ID`, `PLATFORM_VAULT_APPROLE_SECRET_ID_SECRET_REF`, cache TTL/staleness settings, retry settings, and lease renewal threshold.

`CONNECTOR_VAULT_MODE` remains as a one-release compatibility alias and is deprecated for removal in v1.4.0. Imports of `SecretProvider` from `notifications/channel_router.py` are also deprecated; use `platform.common.secret_provider`.

Operators can migrate with `platform-cli vault migrate-from-k8s`, verify with `platform-cli vault verify-migration`, and use `platform-cli vault status` as the operator-facing fallback until the UPD-036 `/admin/security/vault` page lands.

## OAuth Bootstrap Notes

New provider bootstrap variables include `PLATFORM_OAUTH_GOOGLE_ENABLED`, `PLATFORM_OAUTH_GOOGLE_CLIENT_ID`, `PLATFORM_OAUTH_GOOGLE_CLIENT_SECRET`, `PLATFORM_OAUTH_GOOGLE_CLIENT_SECRET_FILE`, `PLATFORM_OAUTH_GOOGLE_REDIRECT_URI`, `PLATFORM_OAUTH_GOOGLE_ALLOWED_DOMAINS`, `PLATFORM_OAUTH_GOOGLE_GROUP_ROLE_MAPPINGS`, `PLATFORM_OAUTH_GOOGLE_REQUIRE_MFA`, `PLATFORM_OAUTH_GOOGLE_DEFAULT_ROLE`, `PLATFORM_OAUTH_GOOGLE_FORCE_UPDATE`, and the equivalent `PLATFORM_OAUTH_GITHUB_*` keys using `ALLOWED_ORGS` and `TEAM_ROLE_MAPPINGS`.

The legacy `OAUTH_SECRET_*` fallback is removed. Runtime OAuth secret resolution now delegates to `SecretProvider`, and bootstrap writes client secrets to Vault KV v2 paths under `secret/data/musematic/{environment}/oauth/{provider}/client-secret`.

Database migration `069_oauth_provider_env_bootstrap.py` adds provider source and edit metadata columns plus per-provider rate-limit storage. Admin OAuth endpoints add rotate-secret, reseed-from-env, history, rate-limits, and status operations. `platform-cli admin oauth export|import` supports GitOps promotion with SHA-256 manifests and Vault path validation.
