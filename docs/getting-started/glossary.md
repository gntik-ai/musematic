# Glossary

| Term | Definition |
| --- | --- |
| Agent | A registered executable capability with identity, metadata, policy, and runtime behavior. |
| Agent FQN | Fully qualified agent name, usually `namespace.name`, used for stable references. |
| Agent revision | A versioned package or metadata snapshot for an agent. |
| ATE | Automated test and evaluation pipeline for semantic and safety scenarios. |
| Bounded context | A service domain with its own model, APIs, events, and ownership. |
| Certification | Trust workflow that approves an agent or revision for broader use. |
| Consumer | User who discovers, invokes, and observes agents. |
| Creator | User who registers, packages, and publishes agents. |
| Enforcer | Policy runtime component that blocks or allows an action. |
| Evaluation | Measurement of quality, reliability, safety, or regression behavior. |
| Execution | Runtime instance of a workflow or agent task. |
| Fleet | Coordinated group of agents with membership, lead, policies, and orchestration rules. |
| FQN | Fully qualified name; the stable identifier for agent references. |
| GID | Goal identifier used to correlate conversations, workflows, traces, and audit events. |
| Governance | Policies and checks that control behavior before or during execution. |
| Goal | Workspace objective that groups related interactions and execution. |
| Guardrail | Safety or policy check applied to inputs, outputs, tools, or model selection. |
| Incident | Operational event that needs triage, runbook execution, and post-mortem tracking. |
| Judge | Evaluation component that scores quality, safety, or correctness. |
| MCP | Model Context Protocol integration surface for tools and context. |
| Observer | Component or user that watches events, metrics, traces, or policy outcomes. |
| Policy bundle | Compiled enforcement rules used by runtime checks. |
| Purpose | Declared reason an agent or tool is allowed to act. |
| Reasoning trace | Structured record of reasoning-engine steps and branch decisions. |
| Registry | Source of truth for agent profiles, revisions, visibility, and lifecycle. |
| Runbook | Operator procedure with symptom, diagnosis, remediation, and verification steps. |
| Sandbox | Isolated runtime used for code execution and artifact capture. |
| Super admin | Platform-wide administrator with emergency and global configuration authority. |
| Tenant | Organizational boundary for identity, policy, data, and configuration. |
| Tool gateway | Controlled path for agents to invoke external tools. |
| Trust radar | UI and data model that summarizes certification and trust dimensions. |
| Visibility grant | Rule that determines who can discover or invoke an agent. |
| Workspace | Collaboration boundary for users, agents, goals, and policies. |
| Workspace goal | Business or operational objective that anchors GID correlation. |
| Zero trust visibility | Default-deny discovery and invocation model unless grants allow access. |
