# Implementation Plan: Agent Marketplace UI

**Branch**: `035-agent-marketplace` | **Date**: 2026-04-12 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/035-agent-marketplace-ui/spec.md`

## Summary

Agent Marketplace UI — a Next.js frontend feature providing natural-language search with debounced filtering, faceted sidebar, paginated agent card grid with infinite scroll, full agent detail pages, side-by-side comparison, ratings/reviews, workspace-scoped invocation flow, recommendation carousel, and creator analytics. Entirely frontend — no new backend services, consumes 9 existing REST endpoints from the `marketplace/` bounded context (feature 021 + adjacent APIs).

## Technical Context

**Language/Version**: TypeScript 5.x, React 18+, Next.js 14+ App Router  
**Primary Dependencies**: shadcn/ui (ALL UI primitives), Tailwind CSS 3.4+, TanStack Query v5, Zustand 5.x, React Hook Form 7.x + Zod 3.x, Recharts 2.x, date-fns 4.x, Lucide React, MSW (dev/test mocks)  
**Storage**: N/A (frontend only — consumes existing backend APIs)  
**Testing**: Vitest + React Testing Library + Playwright + MSW  
**Target Platform**: Web (desktop + mobile responsive)  
**Project Type**: Next.js App Router pages + component library  
**Performance Goals**: Search debounce 300ms; infinite scroll fetchNextPage ≤200ms perceived; initial page load follows existing app scaffold targets  
**Constraints**: No custom CSS files (Tailwind only); no alternative component libraries (shadcn only); no class components; no localStorage for comparison state (session-scoped Zustand only)  
**Scale/Scope**: 7 user stories, 29 FRs, 11 SCs, 3 pages, 18 components, 5 TanStack Query hooks, 1 Zustand store

## Implementation Phases

### Phase 0 — Foundation: Types, Schemas, Store, Hooks

**Goal**: Shared infrastructure used by all subsequent phases — no UI yet.

Files:
- `apps/web/lib/types/marketplace.ts` — All TypeScript types (AgentCard, AgentDetail, AgentReview, MarketplaceSearchParams, etc.)
- `apps/web/lib/schemas/marketplace.ts` — Zod validation schemas (ReviewSubmissionSchema, InvocationSchema, SearchParamsSchema)
- `apps/web/lib/stores/use-comparison-store.ts` — Zustand store (selectedFqns[], add/remove/clear/isSelected/canAdd, max 4)
- `apps/web/lib/hooks/use-marketplace-search.ts` — TanStack Query `useInfiniteQuery` (key: `['marketplace', 'search', params]`)
- `apps/web/lib/hooks/use-agent-detail.ts` — TanStack Query `useQuery` (key: `['marketplace', 'agent', ns, name]`)
- `apps/web/lib/hooks/use-agent-reviews.ts` — `useQuery` + `useSubmitReview` + `useEditReview` mutations
- `apps/web/lib/hooks/use-recommendations.ts` — `useQuery` (key: `['marketplace', 'recommendations']`)
- `apps/web/lib/hooks/use-creator-analytics.ts` — `useQuery` (key: `['marketplace', 'analytics', fqn, periodDays]`)

### Phase 1 — US1: Search, Filter, Sort, Infinite Scroll

**Goal**: `/marketplace` page fully functional with search, faceted filters, sorted results, and infinite scroll.

Files:
- `apps/web/components/features/marketplace/marketplace-search-bar.tsx` — `role="search"`, `aria-label="Search agents"`, `useDebouncedValue(300ms)`, URL push on change
- `apps/web/components/features/marketplace/filter-sidebar.tsx` — Inline desktop / Sheet mobile, `role="navigation" aria-label="Agent filters"`, `useMediaQuery('(max-width: 640px)')`
- `apps/web/components/features/marketplace/agent-card.tsx` — Name, short description, maturity badge (`aria-label="Maturity: X"`), trust badge (`aria-label="Trust tier: X"`), StarRating, cost indicator, Compare toggle button
- `apps/web/components/features/marketplace/agent-card-grid.tsx` — Responsive CSS grid, loading skeleton, empty state, IntersectionObserver for `onLoadMore`
- `apps/web/components/features/marketplace/comparison-floating-bar.tsx` — Fixed-position, `role="status" aria-live="polite"`, visible ≥1 selected, "Compare now" navigates to `/marketplace/compare`
- `apps/web/components/features/marketplace/star-rating.tsx` — Display-only, `rating: number | null`, renders "No ratings" when null
- `apps/web/app/(main)/marketplace/page.tsx` — Reads `useSearchParams()`, wires all components

### Phase 2 — US2: Agent Detail Page

**Goal**: `/marketplace/[namespace]/[name]` fully functional with all sections.

Files:
- `apps/web/components/features/marketplace/trust-signals-panel.tsx` — Tier badge, certification badges (issue/expiry dates), evaluation summary, tier progression timeline
- `apps/web/components/features/marketplace/agent-revisions.tsx` — Chronological version list, current version highlighted
- `apps/web/components/features/marketplace/policy-list.tsx` — Policy name, type, enforcement badge
- `apps/web/components/features/marketplace/quality-metrics.tsx` — Eval score, robustness score, pass rate, last evaluated date
- `apps/web/components/features/marketplace/agent-detail.tsx` — Tabs: Overview | Policies | Quality Metrics | Reviews | Analytics (owner only, not rendered in DOM for non-owners); "Start Conversation" button (disabled + tooltip when `!visibleToCurrentUser`)
- `apps/web/app/(main)/marketplace/[namespace]/[name]/page.tsx` — Calls `useAgentDetail`, handles archived agent ("This agent is no longer available" state)

### Phase 3 — US3: Comparison

**Goal**: `/marketplace/compare` with side-by-side table, max 4 agents, N/A for missing values.

Files:
- `apps/web/components/features/marketplace/comparison-view.tsx` — Table layout (agents as columns, attributes as rows), N/A indicator `"—"`, remove agent button
- `apps/web/app/(main)/marketplace/compare/page.tsx` — Reads `?agents=` CSV param, calls `useAgentDetail` for each FQN, back-link preserves search state

### Phase 4 — US4: Ratings & Reviews

**Goal**: Reviews list, submit form, edit own review, zero-review empty state.

Files:
- `apps/web/components/features/marketplace/star-rating-input.tsx` — 5 `<button>` elements, `aria-label="Rate N out of 5 stars"`, arrow key support, `aria-current="true"` on selected
- `apps/web/components/features/marketplace/reviews-section.tsx` — Review list (load-more pagination), average + count header, write/edit form (RHF + ReviewSubmissionSchema), optimistic update on submit

### Phase 5 — US5: Invocation Flow

**Goal**: "Start Conversation" → Dialog → workspace selector → task brief → redirect.

Files:
- `apps/web/components/features/marketplace/invoke-agent-dialog.tsx` — shadcn Dialog, two-step flow (workspace Select/RadioGroup → task brief Textarea), auto-skip to step 2 when 1 workspace, `router.push('/conversations/new?agent=...&workspace=...&brief=...')` on confirm

### Phase 6 — US6: Recommendation Carousel

**Goal**: Carousel visible ≥3 recommendations; fallback header text for popular/trending.

Files:
- `apps/web/components/features/marketplace/recommendation-carousel.tsx` — shadcn ScrollArea + `flex flex-nowrap`, `role="region" aria-label="Recommended agents"`, compact `AgentCard` variant, header text from `reason` field, renders nothing when `agents.length < 3`

### Phase 7 — US7: Creator Analytics

**Goal**: Analytics tab for agent owners with usage chart, satisfaction trend, common failures.

Files:
- `apps/web/components/features/marketplace/usage-chart.tsx` — Recharts BarChart, `stroke`/`fill` from `hsl(var(--chart-1))` tokens
- `apps/web/components/features/marketplace/satisfaction-trend-chart.tsx` — Recharts LineChart, 5.0 max y-axis, `hsl(var(--chart-2))` token
- `apps/web/components/features/marketplace/creator-analytics-tab.tsx` — Calls `useCreatorAnalytics`, renders UsageChart + SatisfactionTrendChart + common failures list

### Phase 8 — Polish & Cross-Cutting

**Goal**: Dark mode, keyboard navigation, MSW mocks, test coverage.

- Verify all `dark:` Tailwind variants on badges, charts, star ratings
- Keyboard navigation audit: focus trap in InvokeAgentDialog, arrow keys in StarRatingInput, Enter on AgentCard navigates
- MSW handlers for all 9 marketplace API endpoints (dev + test)
- Vitest + RTL unit tests for all 18 components
- Playwright E2E tests for T01, T07, T10, T13, T16 (primary user flows)
