# Implementation Plan: Simulation Controller

**Branch**: `012-simulation-controller` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)

## Summary

Build the Go Simulation Controller satellite service (`services/simulation-controller/`) that provisions isolated simulation environments in the `platform-simulation` Kubernetes namespace, enforces network isolation via NetworkPolicy, manages simulation lifecycle, streams events to subscribers, collects artifacts to a dedicated MinIO bucket, and supports Accredited Testing Environments (ATEs) with pre-loaded test scenarios.

## Technical Context

**Language/Version**: Go 1.22+  
**Primary Dependencies**: `google.golang.org/grpc 1.67+`, `k8s.io/client-go 0.31+`, `github.com/jackc/pgx/v5`, `github.com/confluentinc/confluent-kafka-go/v2`, `github.com/aws/aws-sdk-go-v2`  
**Storage**: PostgreSQL 16 (simulation records, ATE results), MinIO (artifacts, ATE reports)  
**Testing**: `testing` + `testify 1.9` (table-driven, with race detector)  
**Target Platform**: Linux amd64, Kubernetes `platform-simulation` namespace  
**Project Type**: gRPC satellite service  
**Performance Goals**: Status queries < 100ms (SC-012); event delivery < 500ms (SC-005); simulation creation < 60s (SC-001)  
**Constraints**: Docker image < 50MB (distroless); test coverage ‚â• 95%; gRPC port 50055; zero production egress from simulation pods  
**Scale/Scope**: 20+ concurrent simulations (SC-009)

## Implementation Phases

### Phase 1 ‚Äî Setup & Infrastructure (Unblocks All Stories)
- Go module init with all dependencies at `services/simulation-controller/go.mod`
- Full directory structure with stub files
- Proto generation (`make proto`) from `proto/simulation_controller.proto`
- `pkg/persistence/` adapters: pgx pool, Kafka producer, MinIO client
- `pkg/metrics/metrics.go`: all Prometheus instruments
- `api/grpc/v1/interceptors.go`: OTel, panic recovery, logging chain
- `api/grpc/v1/handler.go` skeleton: `Handler` struct, all 6 methods stubbed Unimplemented
- `cmd/simulation-controller/main.go`: env-driven startup, health service registration, graceful shutdown
- PostgreSQL migration at `services/simulation-controller/migrations/001_initial_schema.sql`
- NetworkPolicy manifest applied to `platform-simulation` namespace at startup (idempotent)

### Phase 2 ‚Äî US1: Simulation Environment Creation
- `internal/sim_manager/state.go`: `SimulationState` struct, `sync.Map` state registry, `RebuildFromPodList()` on startup
- `internal/sim_manager/network_policy.go`: `EnsureNetworkPolicy()` ‚Äî deny-all egress to production namespaces, allow `platform-data` for MinIO/Kafka
- `internal/sim_manager/pod.go`: `CreatePod()` ‚Äî full pod spec with labels, injected env vars (SIMULATION=true, SIMULATION_ID, SIMULATION_BUCKET), security context (UID 65534, drop ALL, read-only rootfs), emptyDir volumes, `activeDeadlineSeconds`
- `internal/event_streamer/fanout.go`: fan-out registry with `sync.RWMutex`-protected `map[string][]chan SimulationEvent`
- `api/grpc/v1/handler.go`: `CreateSimulation` method ‚Äî call SimManager, publish CREATED event, return handle
- Unit tests: pod spec labels/env/security, state map registration, NetworkPolicy spec correctness

### Phase 3 ‚Äî US2: Simulation Monitoring and Termination
- `internal/sim_manager/pod.go`: `DeletePod()` with `GracePeriodSeconds=10`
- `internal/sim_manager/orphan_scanner.go`: 60s goroutine, list pods by `simulation=true` label, delete orphans not in state map
- `api/grpc/v1/handler.go`: `GetSimulationStatus` (read from in-memory map, compute elapsed) and `TerminateSimulation` (delete pod, update DB, publish TERMINATED event)
- PostgreSQL writes: `UPDATE simulations SET status=..., terminated_at=... WHERE simulation_id=...`
- Unit tests: status read < 1ms from map, termination cleanup sequence (pod deleted ‚Üí DB updated ‚Üí event published ‚Üí fanout closed), multiple simulations unaffected by single termination

### Phase 4 ‚Äî US3: Artifact Collection
- `internal/artifact_collector/exec.go`: SPDY remotecommand exec, `tar -czf - {path}` pipe to MinIO `simulation-artifacts/{simulation_id}/{filename}.tar.gz`; MinIO `PutObject` with `Metadata: {"x-amz-meta-simulation": "true", "x-amz-meta-simulation-id": simulation_id}`; insert `simulation_artifacts` row
- `api/grpc/v1/handler.go`: `CollectSimulationArtifacts` method; publish ARTIFACT_COLLECTED event; set `partial=true` if pod no longer exists
- Unit tests: metadata tags present, artifact object_key format, partial=true when pod terminated

### Phase 5 ‚Äî US4: Simulation Event Streaming
- `internal/event_streamer/pod_watch.go`: `client-go` `Watch()` on `platform-simulation` namespace with `labelSelector=simulation-id={id}`; convert `watch.Event` types to `SimulationEvent` (ADDED‚ÜíPOD_CREATED, MODIFIED‚ÜíPOD_RUNNING/COMPLETED/FAILED/OOM, DELETED‚ÜíTERMINATED); detect OOM via `containerStatuses.lastState.terminated.reason=OOMKilled`
- `internal/event_streamer/streamer.go`: start pod Watch goroutine + subscribe fan-out channel; fan both to client channel; produce all events to Kafka `simulation.events` keyed by `simulation_id`
- `api/grpc/v1/handler.go`: `StreamSimulationEvents` ‚Äî subscribe, forward to gRPC stream, close on terminal event
- Unit tests: simulation=true on every event, stream close on TERMINATED, Kafka producer called per event

### Phase 6 ‚Äî US5: Accredited Testing Environment
- `internal/ate_runner/configmap.go`: build `v1.ConfigMap` with `scenarios.json` (JSON-encoded scenarios array) and `dataset_refs.json`; `client-go` create in `platform-simulation` namespace
- `internal/ate_runner/results.go`: consume ATE_SCENARIO_COMPLETED events from fan-out registry; parse per-scenario results from event metadata; insert `ate_results` rows; after all scenarios complete, generate report JSON, upload to `simulation-artifacts/{simulation_id}/ate-report.json`, update `ate_sessions.report_object_key`
- `api/grpc/v1/handler.go`: `CreateAccreditedTestEnv` ‚Äî create ConfigMap, create simulation pod with ATE mounts and env vars, insert `simulations` and `ate_sessions` rows, return ATEHandle
- Unit tests: ConfigMap content JSON correctness, ATE report schema, scenario result aggregation (passed/failed counts)

### Phase 7 ‚Äî Polish & Cross-Cutting
- `Dockerfile`: golang:1.22-alpine builder + `gcr.io/distroless/static:nonroot`; verify `docker images` < 50MB
- Helm chart at `deploy/helm/simulation-controller/`: Deployment, Service (port 50055), ClusterRole (scoped to `platform-simulation`: pods create/delete/list/watch, configmaps create/delete, networkpolicies create), ServiceAccount, ClusterRoleBinding, HPA (min=1, max=5, targetCPU=70%)
- `.golangci.yml` at `services/simulation-controller/.golangci.yml`

## Key Decisions (from research.md)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Service structure | Go binary, port 50055, `platform-simulation` | Constitution gRPC Registry |
| Pod isolation | Namespace + NetworkPolicy + label enforcement | ¬ßVII mandate; defense in depth |
| State management | In-memory sync.Map + PostgreSQL (no Redis) | Coarse-grained state; no sub-ms requirement |
| Artifact collection | remotecommand tar ‚Üí MinIO `simulation-artifacts` | Proven pattern from sandbox-manager (010) |
| Event streaming | Server-streaming gRPC + pod Watch + Kafka `simulation.events` | Push-driven; constitution topic registry |
| ATE implementation | Specialized simulation with ConfigMap-injected scenarios | Reuses isolation logic; Kubernetes-native |
| Orphan detection | 60s label-based pod scan | Same pattern as sandbox-manager (010) |
| gRPC methods | 6 RPCs | Cleanly maps to 5 user stories |
