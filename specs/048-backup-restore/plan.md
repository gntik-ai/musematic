# Implementation Plan: Backup and Restore

**Branch**: `048-backup-restore` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)  
**Input**: Unified backup orchestration across all 8 data stores with verification, restore, and operator CLI integration.

## Summary

Feature 045 scaffolded a working but incomplete backup subsystem in `apps/ops-cli/src/platform_cli/backup/`. This feature completes it by: adding the Kafka consumer-group-offset store (8th store), wiring the existing distributed lock into backup create, enforcing dependency ordering in backup/restore, adding auto-tag generation with format validation, exposing a dedicated `verify` subcommand, adding per-store duration tracking and schema versioning to the manifest, adding a `BackupScheduler` for automated scheduled backups with retention, and writing comprehensive unit + integration tests.

## Technical Context

**Language/Version**: Python 3.12+  
**Primary Dependencies**: Typer 0.12+, Rich, Pydantic v2, APScheduler 3.x, aiokafka 0.11+, aioboto3 (optional, for S3 upload), pytest + pytest-asyncio 8.x  
**Storage**: JSON manifests on local filesystem or MinIO bucket (no database tables)  
**Testing**: pytest + pytest-asyncio, monkeypatch for subprocess/network isolation  
**Target Platform**: Linux (Kubernetes + local/Docker deployment modes)  
**Project Type**: CLI tool (extends `apps/ops-cli/`)  
**Performance Goals**: Full backup ≤30 min on 50 GB total data; restore ≤45 min  
**Constraints**: Backup operations run sequentially per store (not concurrent). Lock prevents concurrent backup runs.  
**Scale/Scope**: 8 data stores, 6 CLI subcommands, 1 scheduler class

## Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Kafka backup tool | `aiokafka.AIOKafkaAdminClient` | No native CLI required; offsets only per spec |
| Backup order | postgresql → qdrant → neo4j → clickhouse → redis → opensearch → kafka → minio | System of record first |
| Restore order | Reverse of above (minio first) | MinIO refs must exist before PostgreSQL |
| Auto-tag format | `backup-{YYYYMMDD-HHMMSS}-{NNN}` | Spec FR-005 mandated |
| Lock name | `platform-backup-lock` | Distinct from installer lock |
| S3 upload | Conditional on `backup_storage` URL scheme | Local-first; cloud optional |
| Scheduling | APScheduler 3.x `BlockingScheduler` | Constitution stack; portable |
| Confirmation | `typer.confirm()` + `--yes` bypass | FR-011 + automation support |

## Implementation Phases

### Phase 1: Prerequisites — Model Extensions (US1 foundation)

Extend existing models and `build_artifact()` helper before any other changes.

**Files**:
- `apps/ops-cli/src/platform_cli/models.py` — add `duration_seconds: float = 0.0` to `BackupArtifact`; add `schema_version: int = 1` and `total_duration_seconds: float = 0.0` to `BackupManifest`; bump `CURRENT_SCHEMA_VERSION = 1` constant
- `apps/ops-cli/src/platform_cli/backup/stores/common.py` — add `duration_seconds: float = 0.0` parameter to `build_artifact()`

**Test**: `test_backup.py` — verify model serialisation/deserialisation still round-trips; verify `build_artifact()` accepts optional duration.

---

### Phase 2: Kafka Store (US1 — 8th store)

Implement the 8th store for consumer group offset backup/restore.

**Files**:
- `apps/ops-cli/src/platform_cli/backup/stores/kafka.py` — NEW  
  - `KafkaBackup(bootstrap_servers: str)` 
  - `async def backup(output_dir) -> BackupArtifact`: list consumer groups + their offsets via `aiokafka.AIOKafkaAdminClient`, serialise to `kafka-offsets.json`
  - `async def restore(artifact_path) -> bool`: load JSON, commit offsets back via `AIOKafkaAdminClient`
  - Graceful skip if no consumer groups exist (returns empty offsets artifact)

**Test**: `test_backup.py` — mock `aiokafka.AIOKafkaAdminClient`; assert artifact store="kafka"; assert restore commits offsets.

---

### Phase 3: Orchestrator Hardening (US1, US2 — core backup/restore)

Wire all new capabilities into the orchestrator.

**Files**:
- `apps/ops-cli/src/platform_cli/backup/orchestrator.py`:
  - Add Kafka store to `_stores()` (both local and k8s modes)
  - Add `BACKUP_ORDER` and `RESTORE_ORDER` class-level constants (ordered lists)
  - Iterate stores in `BACKUP_ORDER` in `create()`
  - Iterate artifacts in `RESTORE_ORDER` in `restore()`
  - Auto-tag generation: if `tag is None`, generate `backup-{YYYYMMDD-HHMMSS}-{NNN}`
  - Tag validation: if custom tag, validate against `^[a-zA-Z0-9_-]+$` (raise `ValueError` early)
  - Duration tracking: `time.monotonic()` around each store's `backup()` call; pass `duration_seconds` to `build_artifact()`
  - Total duration: `completed_at` minus `created_at` for `total_duration_seconds` on manifest
  - Lock acquisition/release: acquire `platform-backup-lock` before first store, release in `finally`
  - Rich `Progress` context: per-store rows with spinner → ✓/✗, elapsed time; suppressed when `headless=True`
  - S3 upload: if `storage_root` is an S3/MinIO URL, upload artifacts after local creation
  - Remove `stores_filter` parameter from `create()` method signature (full backup only per spec)
  - Schema version check in `restore()`: if manifest `schema_version > CURRENT_SCHEMA_VERSION`, raise `RuntimeError`
  
- `apps/ops-cli/src/platform_cli/backup/manifest.py`:
  - Ensure `save()` and `create()` write `schema_version` and `total_duration_seconds`

**Test**: `test_backup.py` — test auto-tag generation, tag validation error, lock acquisition/release, dependency order, schema version check.

### Phase 4: CLI Commands (US1, US2, US3, US4, US5)

Update and add CLI subcommands.

**Files**:
- `apps/ops-cli/src/platform_cli/commands/backup.py`:
  - `create_backup`: remove `stores` parameter, add Rich progress output
  - `restore_backup`: add `--yes/-y` flag and `typer.confirm()` before proceeding; show store list in confirmation
  - `list_backups`: update table columns (add size-human-readable, store count)
  - Add `verify_backup(backup_id: str)` subcommand: calls `orchestrator.restore(backup_id, verify_only=True)`; separate exit code on failure

**Test**: `test_backup.py` + `test_commands_more.py` — test `--yes` bypass, confirmation prompt cancel, verify exit code on mismatch.

### Phase 5: Scheduler (US6 — automated scheduled backups)

Implement scheduled backup with retention.

**Files**:
- `apps/ops-cli/src/platform_cli/backup/scheduler.py` — NEW:
  - `BackupScheduler(config: InstallerConfig, storage_root: Path | None = None)`
  - `run_once(retention_days: int = 30) -> ScheduledBackupResult`: creates a backup tagged `scheduled-{YYYYMMDD-HHMMSS}-{NNN}`, then prunes manifests older than `retention_days`
  - `start(cron_expression: str, retention_days: int = 30)`: `BlockingScheduler` firing `run_once()` per cron; logs each run result
  - `_prune(retention_days: int) -> int`: returns count of deleted manifests
  
- `apps/ops-cli/src/platform_cli/commands/backup.py`:
  - Add `schedule_app = typer.Typer()` sub-app
  - `backup schedule start --cron --retention-days --storage-location`
  - `backup schedule run-once --retention-days --storage-location`
  - Register `schedule_app` on `backup_app`

**Test**: `test_backup_scheduler.py` — test `run_once()` with FakeStore, `_prune()` deletes old manifests, concurrent-lock prevention.

Round-trip tests that exercise the full backup → verify → restore flow.
