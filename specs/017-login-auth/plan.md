# Implementation Plan: Login and Authentication UI

**Branch**: `017-login-auth` | **Date**: 2026-04-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/017-login-auth/spec.md`

## Summary

Build the login and authentication UI for the Agentic Mesh Platform frontend (`apps/web/`). Delivers: email/password login form with inline validation, two-step MFA verification (TOTP + recovery code toggle), real-time lockout countdown from backend-provided duration, password reset flow (forgot-password + reset completion with per-rule strength validation), and MFA enrollment dialog (QR code + verification + recovery codes with mandatory acknowledgment). All pages live in the `app/(auth)/` route group (no app shell). The MFA enrollment dialog is a modal overlay in `app/(main)/layout.tsx`. Uses React Hook Form + Zod, TanStack Query mutations, and the auth Zustand store from feature 015.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20+  
**Primary Dependencies**: Next.js 14+ App Router, React 18+, shadcn/ui (Form, Input, Button, Dialog, InputOTP), React Hook Form 7.x + Zod 3.x, TanStack Query v5, Zustand 5.x (existing auth store), qrcode.react, date-fns 4.x, Lucide React  
**Storage**: Zustand auth store (access token in memory, refresh token in localStorage) — inherited from feature 015  
**Testing**: Vitest + React Testing Library, MSW 2.x (API mocking), Playwright (E2E)  
**Target Platform**: Browser (modern); Next.js App Router with SSR-safe components  
**Project Type**: Feature bounded context within Next.js frontend application  
**Performance Goals**: Login page loads and is interactive within 2 seconds; lockout countdown updates every second with no lag  
**Constraints**: shadcn/ui for ALL UI primitives; Tailwind CSS for ALL styling; no new global Zustand stores (login flow uses local state); pnpm 9+  
**Scale/Scope**: 5 user stories, 17 FRs, 12 UI contracts, 6 TanStack Query mutations, 2 custom hooks

## Implementation Phases

### Phase 1 — Setup & Foundations

- Add `qrcode.react` dependency (`pnpm add qrcode.react @types/qrcode.react`)
- Add shadcn `InputOTP` component if not present (`pnpm dlx shadcn@latest add input-otp`)
- Create `lib/schemas/auth-schemas.ts` — all Zod schemas (login, forgotPassword, resetPassword, mfaCode, recoveryCode)
- Create `lib/api/auth.ts` — all TypeScript request/response interfaces + API call functions using `lib/api.ts` from feature 015
- Create `lib/hooks/use-auth-mutations.ts` — 6 TanStack Query `useMutation` hooks
- Create `lib/hooks/use-lockout-countdown.ts` — countdown hook with `useEffect` + `setInterval`
- Create `app/(auth)/layout.tsx` — minimal centered layout (no sidebar, no header, brand logo only)

### Phase 2 — US1: Email/Password Login Form

- `components/features/auth/login-form/LoginForm.tsx` — RHF + Zod form; email/password; submit fires `useLoginMutation`; handles 401 (generic error), 429 (pass lockout data to parent), MFA challenge (pass session token to parent)
- `app/(auth)/login/page.tsx` — owns `LoginFlowState` discriminated union; renders `<LoginForm>`, `<MfaChallengeForm>`, or `<LockoutMessage>` based on step; reads `?redirectTo` param; calls `authStore.setAuth()` on success; calls `router.push(redirectTo ?? '/dashboard')`
- Dark mode: verify all elements use CSS var tokens
- Keyboard nav: verify Tab order and Enter submission

### Phase 3 — US2: TOTP MFA Verification Step

- `components/features/auth/login-form/MfaChallengeForm.tsx` — InputOTP or single numeric input; auto-submit at 6 digits; recovery code toggle; fires `useMfaVerifyMutation` with `session_token` from parent state
- Wire to `LoginPage` flow state: on success → `authStore.setAuth()` + redirect; on error → clear input + show error message
- Recovery code consumed toast: show after redirect if `recovery_code_consumed: true`

### Phase 4 — US3: Account Lockout Feedback

- `components/features/auth/login-form/LockoutMessage.tsx` — renders countdown from `use-lockout-countdown` hook; disables form interactions
- `use-lockout-countdown.ts` — `useEffect` sets up `setInterval(1000)`; computes `remaining = Math.max(0, unlockAt - Date.now())`; calls `onExpired()` when remaining hits 0; cleans up interval on unmount
- Wire to `LoginPage`: when `LoginFlowState.step === 'locked'`, render `<LockoutMessage>`; `onExpired` resets to `credentials` step

### Phase 5 — US4: Password Reset Flow

- `app/(auth)/forgot-password/page.tsx` — renders `<ForgotPasswordForm>`
- `components/features/auth/password-reset/ForgotPasswordForm.tsx` — email input; fires `useForgotPasswordMutation`; always shows confirmation on API response (regardless of status); never reveals if email is registered
- `app/(auth)/reset-password/[token]/page.tsx` — reads token from path; renders `<ResetPasswordForm token={token}>`; handles `TOKEN_EXPIRED`/`TOKEN_ALREADY_USED` error state
- `components/features/auth/password-reset/ResetPasswordForm.tsx` — new password + confirm password; real-time per-rule strength indicators; fires `useResetPasswordMutation`; on success: `router.push('/login?message=password_updated')`
- Login page reads `?message=password_updated` search param → shows success toast

### Phase 6 — US5: MFA Enrollment Dialog

- `components/features/auth/mfa-enrollment/QrCodeStep.tsx` — fires `useMfaEnrollMutation` on mount; shows QR code via `<QRCodeSVG>` from `qrcode.react`; shows secret key text fallback
- `components/features/auth/mfa-enrollment/VerificationStep.tsx` — InputOTP; fires `useMfaConfirmMutation`; on success advances to recovery codes step
- `components/features/auth/mfa-enrollment/RecoveryCodesStep.tsx` — displays recovery codes; "Copy all" button; acknowledgment checkbox; "Complete setup" disabled until acknowledged; prevents dialog close without acknowledgment
- `components/features/auth/mfa-enrollment/MfaEnrollmentDialog.tsx` — `Dialog` with controlled `open` (no close button); routes between steps via local state; closes only after `RecoveryCodesStep` completes
- Modify `app/(main)/layout.tsx` — add `{!user.mfaEnrolled && <MfaEnrollmentDialog />}` render

### Phase 7 — Polish & Cross-Cutting Concerns

- Add E2E Playwright test for the full login → dashboard flow
- Add E2E test for the MFA enrollment flow
- Verify 100% keyboard navigability (Tab order, Enter submit, Esc behavior on dialogs)
- Verify dark mode on all auth pages and dialogs
- Verify responsive layout 320px → 2560px (no horizontal scroll, no overlapping)
- TypeScript strict clean run (`tsc --noEmit`)
- ESLint clean run (`pnpm lint`)
- Test coverage audit (target ≥ 95% for auth components)

## Key Decisions (from research.md)

1. **Route group**: Auth pages in `app/(auth)/` (no shell) — NOT `app/(main)/`; enrollment dialog in `(main)` layout
2. **Login flow state**: Local `LoginFlowState` discriminated union in `LoginPage` — not Zustand
3. **MFA code input**: `InputOTP` from shadcn/ui (preferred) or single `maxLength=6` input
4. **Lockout countdown**: `useEffect` + `setInterval` from backend `lockout_seconds`; no polling
5. **Password strength**: Zod inline `.regex()` validators matching feature 016 backend rules exactly
6. **QR code**: `qrcode.react` `<QRCodeSVG>` from backend `provisioning_uri`; always show text fallback
7. **Deep link**: `?redirectTo` search param; validate starts with `/` to prevent open redirect
8. **Forms**: React Hook Form + Zod via shadcn Form integration — constitution mandate
9. **Token storage**: Calls `authStore.setAuth()` from feature 015; no new storage mechanism
10. **MFA enrollment**: Modal `Dialog` on `(main)` layout; unblockable until recovery codes acknowledged
11. **API integration**: TanStack Query `useMutation` in `lib/hooks/use-auth-mutations.ts`
