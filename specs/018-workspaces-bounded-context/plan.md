# Implementation Plan: Workspaces Bounded Context

**Branch**: `018-workspaces-bounded-context` | **Date**: 2026-04-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/018-workspaces-bounded-context/spec.md`

## Summary

Build the `workspaces/` bounded context within `apps/control-plane/src/platform/`. This covers workspace CRUD (create, get, list, update, archive, restore, delete), membership management (add, remove, change role, list with role-based authorization), default workspace provisioning via Kafka consumer on `accounts.user.activated`, workspace goals with first-class Goal ID (GID) correlation dimension and status state machine, workspace-wide visibility grants (FQN pattern lists overriding per-agent zero-trust defaults), workspace settings (super-context subscription metadata), per-user workspace limits (read from accounts via in-process service interface), and Kafka event emission on all lifecycle and membership transitions.

## Key Decisions (from research.md)

1. **Data isolation**: `WorkspaceScopedMixin` on cross-context entities; explicit FKs on workspace-internal entities
2. **Workspace roles**: `WorkspaceRole` enum in workspaces context — separate from auth RBAC roles
3. **Default provisioning**: Kafka consumer on `accounts.user.activated` → idempotent `create_default_workspace()`
4. **Workspace limits**: Read from accounts context via in-process `accounts_service.get_user_workspace_limit(user_id)`
5. **Settings storage**: Separate `workspaces_settings` table with PostgreSQL `ARRAY` columns for subscriptions
6. **Visibility grants**: Separate `workspaces_visibility_grants` table with `TEXT[]` arrays; atomic replace
7. **Goal state machine**: Dict-based `VALID_GOAL_TRANSITIONS` in `state_machine.py` — service-layer enforcement
8. **Kafka topic**: `workspaces.events` (new topic — needs constitution registry update)
9. **Permanent delete**: Deferred cleanup via background worker; workspace marked `deleted` immediately
10. **Authorization**: Service-layer role checks; admin cannot promote to owner or demote owner
11. **Name uniqueness**: Per-owner composite unique index on `(owner_id, name, status)` where status != 'deleted'
