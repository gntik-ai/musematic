# Implementation Plan: Next.js Application Scaffold

**Branch**: `015-nextjs-app-scaffold` | **Date**: 2026-04-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/015-nextjs-app-scaffold/spec.md`

## Summary

Initialize the Next.js 14+ App Router frontend application (`apps/web/`) for the Agentic Mesh Platform. Deliver: TypeScript strict mode with path aliases, shadcn/ui component system, Tailwind CSS with CSS-variable dark mode theme, typed fetch API client (`lib/api.ts`) with JWT injection and refresh-on-401, WebSocket client (`lib/ws.ts`) with topic subscriptions and reconnect, Zustand auth + workspace stores, TanStack Query v5 factory hooks, app shell (collapsible sidebar with RBAC-filtered nav, header with workspace selector, breadcrumbs, command palette), and 11 shared UI components (DataTable, StatusBadge, MetricCard, ScoreGauge, EmptyState, ConfirmDialog, CodeBlock, JsonViewer, Timeline, SearchInput, FilterBar).

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20+  
**Primary Dependencies**: Next.js 14+ (App Router), React 18+, shadcn/ui (all primitives), Tailwind CSS 3.4+, TanStack Query v5, TanStack Table v8, Zustand 5.x, Recharts 2.x, next-themes, cmdk (via shadcn Command), highlight.js, date-fns 4.x, Lucide React, Zod 3.x, React Hook Form 7.x  
**Storage**: `localStorage` (Zustand persist middleware for auth refresh token + workspace preference)  
**Testing**: Vitest + React Testing Library, Playwright (E2E), MSW (API mocking)  
**Target Platform**: Browser (modern); SSR-safe (Next.js server components where appropriate)  
**Project Type**: Next.js web application — frontend of modular monolith platform  
**Performance Goals**: First Load JS < 200kB for main route; app shell renders < 3s; command palette opens < 100ms  
**Constraints**: pnpm 9+ (constitution mandate); shadcn/ui for ALL UI primitives (no alternatives); Tailwind CSS for ALL styling (no CSS modules); TypeScript strict + `noUncheckedIndexedAccess`  
**Scale/Scope**: 6 user stories, 11 shared components, 2 Zustand stores, 1 API client factory, 1 WebSocket client, 30+ shadcn primitives

## Implementation Phases

### Phase 1 — Project Setup & Theme Foundation (US1)
- Initialize Next.js 14+ App Router project in `apps/web/` with TypeScript strict mode
- Configure `tsconfig.json`: strict, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, `@/*` path alias
- Configure `tailwind.config.ts`: CSS variable tokens, `dark` class strategy, custom brand tokens
- Write `globals.css`: `:root` + `.dark` CSS custom properties for all shadcn + brand tokens
- Initialize shadcn/ui: run `shadcn init`, install all core primitives (30+ components)
- Configure `next-themes` `ThemeProvider` — SSR-safe, no FOIT
- Create route group structure: `app/(auth)/`, `app/(main)/`
- Implement root `app/layout.tsx` with providers (Query, Theme, WebSocket)
- Create `app/(auth)/login/page.tsx` placeholder
- Create `app/(main)/layout.tsx` with auth guard (redirect to `/login` if not authenticated)
- Create `app/(main)/page.tsx` placeholder dashboard

### Phase 2 — TypeScript Types & Foundational Infrastructure
- Write all types: `types/auth.ts`, `types/workspace.ts`, `types/api.ts`, `types/navigation.ts`, `types/websocket.ts`
- Write `lib/query-client.ts` — export singleton `QueryClient` with default options
- Configure MSW: `mocks/handlers.ts` with auth + workspace mock endpoints; `mocks/browser.ts`
- Configure Vitest: `vitest.config.ts` with jsdom, path aliases, setup file for RTL + MSW
- Configure Playwright: `playwright.config.ts` targeting `http://localhost:3000`

### Phase 3 — US3: API Client
- Implement `lib/api.ts`: `createApiClient(baseUrl)` factory with:
  - Request interceptor: reads `useAuthStore.getState().accessToken`, injects Bearer header
  - Response interceptor: on 401 → calls `lib/auth.ts` `refreshAccessToken()` → retry once
  - Retry: network errors up to 3x with exponential backoff (1s, 2s, 4s)
  - Error normalization: transforms `{"error": {...}}` into `ApiError` instances
  - Typed methods: `get<T>`, `post<T>`, `put<T>`, `patch<T>`, `delete<T>`
- Implement `lib/auth.ts`: `refreshAccessToken()` — calls `/api/v1/auth/refresh` with stored refresh token, updates auth store, returns new token pair; on failure clears auth and redirects to login
- Implement `lib/hooks/use-api.ts`: `useAppQuery<T>`, `useAppMutation<TData, TVars>`, `useAppInfiniteQuery<T>` with sane defaults (staleTime: 30s, gcTime: 5min)

### Phase 4 — US6: State Management Stores
- Implement `store/auth-store.ts`: Zustand store with `setTokens`, `setUser`, `clearAuth` actions; persist middleware partializing to `{ refreshToken }` only
- Implement `store/workspace-store.ts`: Zustand store with `setCurrentWorkspace` (calls `queryClient.invalidateQueries()`), `setWorkspaceList`, `setSidebarCollapsed`; persist middleware for `{ currentWorkspace, sidebarCollapsed }`

### Phase 5 — US4: WebSocket Client
- Implement `lib/ws.ts`: `WebSocketClient` class with:
  - Single `WebSocket` instance, auto-reconnect (exponential backoff: 1s → 2s → 4s → 8s → cap 30s)
  - `subscribe(channel, handler) → unsubscribe` — registry of channel → Set<handler>
  - `send(channel, type, payload)` — serializes to `{ channel, type, payload }` JSON
  - `connectionState` reactive property — emits `stateChange` events
  - On reconnect: re-registers all active subscriptions
- Implement `components/providers/WebSocketProvider.tsx`: creates client on mount, exposes via `useWebSocket()` hook, disconnects on unmount
- Implement `components/layout/header/ConnectionIndicator.tsx`: subscribes to `connectionState`, renders colored dot with appropriate label

### Phase 6 — US2: App Shell & Navigation
- Implement `components/layout/sidebar/nav-config.ts`: `NAV_ITEMS` static array with `requiredRoles`
- Implement `components/layout/sidebar/Sidebar.tsx`: reads nav config, filters by user roles, renders with collapse animation, `aria-current` on active route
- Implement `components/layout/header/Header.tsx`: workspace selector, breadcrumb, notifications placeholder, user menu

- Implement `components/layout/command-palette/CommandPaletteProvider.tsx`: global `keydown` listener for `Cmd+K` / `Ctrl+K`

## Key Decisions (from research.md)

1. **Route groups** `(main)` + `(auth)` — shell separation without URL pollution
2. **CSS custom properties** (HSL) + `next-themes` class strategy — SSR-safe dark mode, no FOIT
3. **Fetch wrapper** over axios — keeps bundle lean; full interceptor pattern for JWT + refresh + retry
4. **WebSocket** topic/channel subscription model — aligns with Kafka channel naming on backend
5. **Zustand persist** — only `refreshToken` to localStorage (access tokens are ephemeral)
6. **TanStack Query singleton** — exported `queryClient` enables imperative invalidation from Zustand workspace store
7. **Static nav config** with `requiredRoles` — client-side UX filter (server enforces actual RBAC)
8. **shadcn Command (cmdk)** — full-featured command palette without reinventing fuzzy search
9. **TanStack Table v8** for DataTable — pairs naturally with TanStack Query, no extra license needed
10. **Vitest + RTL** over Jest — faster, native ESM, same API; MSW for realistic API mocking
