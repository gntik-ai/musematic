# Implementation Plan: Reasoning Engine

**Branch**: `011-reasoning-engine` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)

## Summary

Build the Go Reasoning Engine satellite service (`services/reasoning-engine/`) that provides reasoning mode selection, real-time atomic budget tracking via Redis Lua scripts, chain-of-thought trace coordination with dual persistence, tree-of-thought branch management using goroutine pools, and self-correction convergence detection with a two-sample window.

## Technical Context

**Language/Version**: Go 1.22+  
**Primary Dependencies**: `google.golang.org/grpc 1.67+`, `github.com/redis/go-redis/v9`, `github.com/jackc/pgx/v5`, `github.com/confluentinc/confluent-kafka-go/v2`, `github.com/aws/aws-sdk-go-v2`  
**Storage**: Redis 7 (hot state), PostgreSQL 16 (cold state), MinIO (large payloads)  
**Testing**: `testing` + `testify 1.9` (table-driven, with race detector)  
**Target Platform**: Linux amd64, Kubernetes `platform-execution` namespace  
**Project Type**: gRPC satellite service  
**Performance Goals**: Budget check p99 < 1ms; mode selection < 50ms; threshold events < 100ms  
**Constraints**: Docker image < 50MB (distroless); test coverage ≥ 95%; gRPC port 50052  
**Scale/Scope**: 100+ concurrent reasoning executions, 10 concurrent ToT branches per execution

## Implementation Phases

### Phase 1 — Setup & Infrastructure (Unblocks All Stories)
- Go module init (`services/reasoning-engine/go.mod`)
- Directory structure (all packages with stub files)
- Proto generation (`make proto`)
- `pkg/persistence/` adapters: Redis cluster client, pgx pool, Kafka producer, MinIO client
- `pkg/lua/loader.go`: load both Lua scripts at startup via `SCRIPT LOAD`, store SHA map
- `pkg/metrics/metrics.go`: define all Prometheus instruments
- `cmd/reasoning-engine/main.go`: wire all packages, start gRPC server with interceptors

### Phase 2 — US1: Reasoning Mode Selection
- `internal/mode_selector/heuristic.go`: complexity scorer (word count, keywords, questions)
- `internal/mode_selector/selector.go`: policy override → budget feasibility → heuristic → allocation
- `api/grpc/v1/handler.go` method: `SelectReasoningMode`
- Unit tests: all 6 modes, policy override, budget downgrade

### Phase 3 — US2: Budget Tracking and Enforcement
- `internal/budget_tracker/redis.go`: `AllocateBudget`, `DecrementBudget` (EVALSHA), `GetStatus`
- `internal/budget_tracker/events.go`: fan-out registry (`sync.RWMutex` map), threshold detection (80%/90%/100%)
- `api/grpc/v1/handler.go` methods: `AllocateReasoningBudget`, `GetReasoningBudgetStatus`, `StreamBudgetEvents`
- Integration tests: concurrent decrements (100 goroutines), threshold event timing, TTL expiry

### Phase 4 — US3: Chain-of-Thought Trace Coordination
- `internal/cot_coordinator/pipeline.go`: stream reader goroutine, worker pool, buffered channel, MinIO routing for >64KB
- PostgreSQL writes: `reasoning_traces`, `reasoning_events` tables
- Kafka emission: `runtime.reasoning` topic
- `api/grpc/v1/handler.go` method: `StreamReasoningTrace`
- Unit tests: buffer overflow (drop oldest), large payload routing, ack counting

### Phase 5 — US4: Tree-of-Thought Branch Management
- `internal/tot_manager/branch.go`: bounded semaphore (`chan struct{}`), goroutine lifecycle, `context.CancelCauseFunc` pruning, panic recovery
- `internal/tot_manager/evaluator.go`: quality/cost scoring, tie-breaking (cost → created_at)
- PostgreSQL writes: `tot_branches` table
- `api/grpc/v1/handler.go` methods: `CreateTreeBranch`, `EvaluateTreeBranches`
- Integration tests: auto-pruning, concurrent execution, no-viable-branches path

### Phase 6 — US5: Self-Correction Convergence Detection
- `internal/correction_loop/convergence.go`: `convergence_check.lua` EVALSHA, `budget_decrement.lua` for iteration count/cost
- `internal/escalation/router.go`: Kafka producer to `monitor.alerts`
- PostgreSQL writes: `correction_iterations` table
- `api/grpc/v1/handler.go` methods: `StartSelfCorrectionLoop`, `SubmitCorrectionIteration`
- Unit tests: convergence at iteration 6 per spec example, budget-exceeded termination, escalation event

### Phase 7 — US6: Budget Event Streaming
- Complete `StreamBudgetEvents` with fan-out registry subscription (already partially implemented in Phase 3)
- Multiple concurrent subscriber test, stream close on budget completion
- Integration test: 2 subscribers receive same events in order

### Phase 8 — Polish & Cross-Cutting
- `Dockerfile`: multi-stage (golang:1.22-alpine builder + `gcr.io/distroless/static:nonroot`)
- Helm chart at `deploy/helm/reasoning-engine/` (Deployment, Service, ConfigMap, HorizontalPodAutoscaler)
- Health check: `grpc.health.v1.Health` implementation
- OpenTelemetry trace propagation in all interceptors
- `golangci-lint` configuration and CI pipeline step
- Final coverage check (≥95%)

## Key Decisions (from research.md)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Service structure | Go binary at `services/reasoning-engine/` | Constitution §3.2 explicit |
| Budget atomicity | Redis Lua scripts via EVALSHA | No interleaving; <1ms p99 |
| ToT concurrency | Goroutine per branch + semaphore | Idiomatic Go; independent cancellation |
| CoT trace | Client-streaming gRPC + async worker pool | Decouples ingestion from persistence |
| Mode selection | Rule-based heuristic (6 modes) | Deterministic, fast, testable; ML out of scope |
| Convergence | two-sample window (delta < ε for 2 consecutive) | Less noise-sensitive than single-sample |
| Budget events | Fan-out registry (in-process) | Established pattern from runtime-controller |
| Cold state | PostgreSQL via pgx/v5, 4 tables | Constitution §2.2, §3.3 |
| Code bridge | Reasoning engine tracks budget; runtime calls sandbox directly | No latency added to code execution path |
