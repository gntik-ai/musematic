# Implementation Plan: Trust and Certification Workbench

**Branch**: `043-trust-certification-workbench` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/043-trust-certification-workbench/spec.md`

## Summary

Frontend-only feature delivering a trust governance workbench for trust officers. Five user stories: a paginated, filterable certification queue (US1); a certification detail page with expandable evidence items and approve/reject reviewer form (US2); a 7-dimension Recharts RadarChart trust profile (US3); a drag-and-drop policy attachment panel with inheritance chain display (US4); and a privacy impact analysis panel showing data category compliance (US5). No new npm packages — all required libraries (Recharts RadarChart, shadcn/ui Collapsible/DataTable/Tabs, TanStack Query, Zustand, React Hook Form + Zod) are already installed.

### Phase 2: TanStack Query Hooks

**Goal**: All server state hooks wired to backend API before any UI is built.

**Tasks**:
1. Create `lib/hooks/use-certifications.ts`:
   - `useCertificationQueue(filters: CertificationQueueFilters)` — `GET /trust/certifications` with query params, 300ms debounce for search
   - `useCertification(certId: string)` — `GET /trust/certifications/{certId}`
2. Create `lib/hooks/use-certification-actions.ts`:
   - `useApproveCertification()` — POST activate + POST evidence ref (two-step mutation)
   - `useRevokeCertification()` — POST revoke with reason
   - `useAddEvidenceRef(certId)` — POST evidence item
3. Create `lib/hooks/use-trust-radar.ts`:
   - `useTrustRadar(agentId: string)` — `GET /trust/agents/{agentId}/trust-profile`; falls back to `GET /trust/agents/{agentId}/tier` with mapped data
4. Create `lib/hooks/use-privacy-impact.ts`:
   - `usePrivacyImpact(agentId: string)` — `GET /trust/agents/{agentId}/privacy-impact`
5. Create `lib/hooks/use-policy-catalog.ts`:
   - `usePolicyCatalog(workspaceId: string, search?: string)` — `GET /policies?workspace_id=...&status=active`
6. Create `lib/hooks/use-effective-policies.ts`:
   - `useEffectivePolicies(agentId: string, workspaceId: string)` — `GET /policies/effective/{agentId}?workspace_id=...`; transforms `PolicyRuleProvenance` into `PolicyBinding[]` with derived `source` + `sourceLabel` fields
7. Create `lib/hooks/use-policy-actions.ts`:
   - `useAttachPolicy()` — POST attach with optimistic update; invalidates `effectivePolicies` query on settle
   - `useDetachPolicy()` — DELETE attach/{id} with optimistic removal from binding list

**Key decisions**:
- `useCertificationQueue` uses 300ms debounced search to meet SC-001 (<10s navigation, queue renders within 300ms)
- `useApproveCertification` chains activate → addEvidenceRef; evidence ref failure is non-blocking
- `useDetachPolicy` uses optimistic update: removes binding from cache immediately, rolls back on error
- `useEffectivePolicies` transforms raw `PolicyRuleProvenance` → `PolicyBinding` with source derivation (scope_type → 'direct' | 'workspace' | 'fleet' | 'global')

## Key Design Decisions (Summary)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Certification queue endpoint | `GET /trust/certifications` (assumed global listing) | Per-agent only endpoint is insufficient for workbench queue |
| Route structure | `/trust-workbench` + `/trust-workbench/[certificationId]?tab=` | Tab routing consistent with features 027/041 |
| Trust radar data | `GET /trust/agents/{agentId}/trust-profile` (assumed) | Spec assumption: pre-computed 7-dimension endpoint |
| Privacy impact data | `GET /trust/agents/{agentId}/privacy-impact` (assumed) | Read companion to service-account-only POST assess |
| Approve/reject API | POST activate + POST revoke separately | Existing backend endpoints; no new endpoint needed |
| Drag-and-drop | HTML5 native drag events | No new packages; single drop target |
| Policy inheritance | `GET /policies/effective/{agentId}` | Pre-resolved inheritance chain in one request |
| New packages | None | All capabilities in existing stack |
