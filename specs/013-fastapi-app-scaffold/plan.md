# Implementation Plan: FastAPI Application Scaffold

**Branch**: `013-fastapi-app-scaffold` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)

## Summary

Build the foundational FastAPI application scaffold for the Python control plane at `apps/control-plane/src/platform/`. This includes the app factory with lifespan hooks, Pydantic Settings, async SQLAlchemy with 6 mixins, Kafka event infrastructure (producer, consumer, envelope, DLQ, registry), correlation ID and JWT auth middleware, client wrappers for 8 data stores and 4 gRPC satellite services, exception hierarchy, pagination helpers, and 8 runtime profile entrypoints.

## Technical Context

**Language/Version**: Python 3.12+  
**Primary Dependencies**: FastAPI 0.115+, Pydantic v2, SQLAlchemy 2.x (async), aiokafka 0.11+, redis-py 5.x (async), grpcio 1.65+, httpx 0.27+, PyJWT 2.x, opentelemetry-sdk 1.27+  
**Storage**: PostgreSQL 16 (via asyncpg), Redis 7 (via redis.asyncio), Kafka, Qdrant, Neo4j, ClickHouse, OpenSearch, MinIO  
**Testing**: pytest 8.x + pytest-asyncio  
**Target Platform**: Linux server, Kubernetes `platform-control` namespace  
**Project Type**: Web service (modular monolith control plane)  
**Performance Goals**: Health endpoint < 500ms; startup < 30s  
**Constraints**: Test coverage ≥ 95%; all async; ruff + mypy --strict  
**Scale/Scope**: Foundation for 30+ bounded contexts, 8 runtime profiles

## Implementation Phases

### Phase 1 — Setup & Configuration
- `pyproject.toml`: all dependencies per constitution §2.1
- `src/platform/common/config.py`: `PlatformSettings` with all sub-settings
- `src/platform/common/models/base.py`: `Base = DeclarativeBase`
- `src/platform/common/models/mixins.py`: 6 mixins

### Phase 2 — US1: Application Bootstrap and Health
- `src/platform/common/database.py`: `AsyncEngine`, `async_sessionmaker`, pool configuration
- `src/platform/main.py`: app factory with `lifespan` async context manager, degraded mode on store failure
- `src/platform/api/health.py`: `GET /health` returning per-dependency status with latency
- Integrate all client wrappers for health check

### Phase 3 — US2: Configuration and Dependency Injection
- `src/platform/common/dependencies.py`: `get_db`, `get_current_user`, `get_workspace`
- Wire settings into all client initialization via lifespan

### Phase 4 — US3: Database Models and Mixins
- Complete mixin implementations with tests
- `SoftDeleteMixin` default query filter
- `EventSourcedMixin` optimistic locking
- `WorkspaceScopedMixin` query filter

### Phase 5 — US4: Kafka Event Infrastructure
- `src/platform/common/events/envelope.py`: `EventEnvelope`, `CorrelationContext`
- `src/platform/common/events/producer.py`: async producer wrapping aiokafka
- `src/platform/common/events/consumer.py`: consumer group manager
- `src/platform/common/events/registry.py`: type registry with schema validation
- `src/platform/common/events/retry.py`: DLQ handler (3 retries, exponential backoff)

### Phase 6 — US5: Request Middleware
- `src/platform/common/correlation.py`: `CorrelationMiddleware`, `ContextVar`, header propagation
- `src/platform/common/auth_middleware.py`: JWT validation, path exemptions, Redis session check

### Phase 7 — US6: Client Wrappers
- All 10 client wrappers in `src/platform/common/clients/`
- Each: connect(), close(), health_check(), typed methods

### Phase 8 — US7: Exception Hierarchy and Pagination
- `src/platform/common/exceptions.py`: full hierarchy + FastAPI exception handler
- `src/platform/common/pagination.py`: CursorPage, OffsetPage, SQLAlchemy query helpers

### Phase 9 — Polish & Cross-Cutting
- 8 entrypoints in `apps/control-plane/entrypoints/`
- `src/platform/common/telemetry.py`: OpenTelemetry auto + manual instrumentation
- ruff + mypy configuration
- Coverage ≥ 95%

## Key Decisions (from research.md)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| App factory | Lifespan context manager | FastAPI 0.109+ recommended; clean startup/shutdown |
| Configuration | Pydantic Settings with nested models | Constitution §2.1; env var overrides |
| DB engine | AsyncEngine + async_sessionmaker | Constitution: async only, SQLAlchemy 2.x |
| Kafka events | aiokafka + canonical EventEnvelope + DLQ | Constitution §2.1; schema validation prevents drift |
| Client wrappers | Thin async wrappers with health check | No business logic in wrappers; lifecycle only |
| Middleware | Correlation ID + JWT auth | Constitution: correlation everywhere; RS256 JWT |
| Exceptions | PlatformError hierarchy → HTTP codes | Constitution error handling section |
| Pagination | Cursor + offset helpers | Cursor for feeds, offset for admin UIs |
| Profiles | 8 entrypoint scripts | Constitution §3.1: 8 modular monolith profiles |
| Telemetry | OpenTelemetry auto + manual | Constitution §2.1: opentelemetry-sdk 1.27+ |
